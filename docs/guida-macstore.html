Prerequisiti

Assicurati di avere tutto il necessario prima di iniziare.

🖥️ Hardware & OS Mac

  • Mac con macOS Ventura 13.5+ (consigliato Sequoia 15+)
  • Almeno 16 GB RAM consigliati
  • 30 GB di spazio libero su disco
  • Architettura Apple Silicon o Intel x86-64

🛠️ Software Xcode MAUI

  • Xcode 16+ (da Mac App Store) — obbligatorio
  • .NET 9+ SDK con workload maui-maccatalyst
  • Visual Studio 2022 for Mac o VS Code con .NET MAUI extension
  • Rider con plugin MAUI (alternativa)

🍎 Apple Developer Account Portal

  • Account Apple Developer attivo (99 $/anno) ✅ già rinnovato
  • Accesso a developer.apple.com
  • Accesso a appstoreconnect.apple.com

📦 Workload .NET MAUI Mac Catalyst ⚡ Diverso da iOS

  • Verifica installazione:
dotnet workload list
  • Installa il workload macOS/Mac Catalyst:
dotnet workload install maui-maccatalyst
ℹ️
.NET MAUI su macOS usa Mac Catalyst, che porta le app UIKit (iOS/iPadOS) su macOS. Non è lo stesso di un'app AppKit nativa, ma funziona benissimo per RadioE45.

Apple Developer Portal – Certificati e Provisioning ⚡ Diverso da iOS

ℹ️
Vai su developer.apple.com → Certificates, Identifiers & Profiles. Per macOS hai bisogno di due certificati invece di uno solo.

Flusso macOS: App ID → 2 Certificati → Provisioning Profile → Firma + .pkg

Verifica/Registra l'App ID Se hai già registrato com.radio45.app per iOS, puoi usare lo stesso identificatore — è valido per tutte le piattaforme Apple.
Vai in Identifiers → com.radio45.app e verifica che la piattaforma includa macOS. Se hai creato l'App ID solo per iOS, potrebbe essere necessario creare un nuovo identificatore esplicito oppure abilitare macOS nella configurazione esistente.
Genera la CSR con Keychain Access (se non già fatto per iOS) Se hai già generato una CSR per la guida iOS, puoi riutilizzarla (la chiave privata è già nel Keychain). Altrimenti segui gli stessi passi:

Keychain Access → Certificate Assistant → Request a Certificate from a Certificate Authority…
Compila email e nome → Saved to disk → Salva il file .certSigningRequest.
Crea il certificato "Apple Distribution" (per la firma del codice app)
ℹ️
Se hai già il certificato Apple Distribution creato per iOS, puoi usarlo anche per macOS. Salta a questo punto se ce l'hai già nel Keychain.
Vai in Certificates → + → scegli Apple Distribution.
Carica la CSR → scarica distribution.cer → doppio clic per importarlo nel Keychain.
Crea il certificato "Mac Installer Distribution" (per firmare il pacchetto .pkg) ⚡ Solo macOS Questo secondo certificato è esclusivo per macOS e firma il pacchetto di installazione (non il codice dell'app).

Vai in Certificates → + → scegli Mac Installer Distribution.
Carica la stessa CSR → scarica macinstallerdist.cer → doppio clic per importarlo nel Keychain.

Dopo l'importazione, nel Keychain di login avrai due certificati di distribuzione:
  • Apple Distribution: [TuoNome] ([TEAMID])
  • 3rd Party Mac Developer Installer: [TuoNome] ([TEAMID])
Crea il Provisioning Profile "Mac App Store Connect" ⚡ Diverso da iOS Vai in Profiles → + → scegli Mac App Store Connect (non "App Store Connect" che è per iOS).

  • App ID: com.radio45.app
  • Certificate: seleziona il certificato Apple Distribution
  • Name: RadioE45 Mac AppStore
Scarica il file .provisionprofile e fai doppio clic per installarlo in Xcode.
✅
Puoi usare il Automatic Signing di Xcode anche per macOS: apri il progetto, vai in Signing & Capabilities, spunta "Automatically manage signing", seleziona il team e scegli la destinazione Mac Catalyst. Xcode gestirà entrambi i certificati automaticamente.

Configurazione progetto .NET MAUI (Mac Catalyst) ⚡ Diverso da iOS

Apri il file RadioE45.csproj e aggiungi/verifica le proprietà per Mac Catalyst:

<!-- RadioE45.csproj -->
<PropertyGroup>
  <!-- Versione app (stessa usata per iOS, incrementa ad ogni upload) -->
  <ApplicationVersion>1</ApplicationVersion>
  <ApplicationDisplayVersion>1.0.0</ApplicationDisplayVersion>

  <!-- Bundle ID (stesso di iOS) -->
  <ApplicationId>com.radio45.app</ApplicationId>

  <!-- Versione minima macOS per Mac Catalyst -->
  <SupportedOSPlatformVersion Condition="$([MSBuild]::GetTargetPlatformIdentifier('$(TargetFramework)')) == 'maccatalyst'">13.1</SupportedOSPlatformVersion>
</PropertyGroup>

<!-- Firma per la distribuzione Mac App Store -->
<PropertyGroup Condition="$(TargetFramework.Contains('-maccatalyst')) and '$(Configuration)' == 'Release'">
  <!-- Certificato per firma del codice -->
  <CodesignKey>Apple Distribution: [TuoNome] ([TEAMID])</CodesignKey>
  <CodesignProvision>RadioE45 Mac AppStore</CodesignProvision>
  <CodesignEntitlements>Platforms/MacCatalyst/Entitlements.plist</CodesignEntitlements>
  <!-- Certificato per firma del pacchetto .pkg -->
  <PackageSigningKey>3rd Party Mac Developer Installer: [TuoNome] ([TEAMID])</PackageSigningKey>
</PropertyGroup>
⚠️
Il valore di ApplicationVersion deve essere incrementato ad ogni caricamento su App Store Connect, anche se la versione visibile rimane uguale. Vale sia per iOS che per macOS — se le pubblichi insieme, incrementa una volta sola.

📁 Verifica Info.plist macOS (Platforms/MacCatalyst/Info.plist) ⚡ Diverso da iOS

Crea o verifica che esista la cartella Platforms/MacCatalyst/ con il suo Info.plist:

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "...">
<plist version="1.0">
<dict>
  <!-- Necessario per Mac App Store -->
  <key>LSMinimumSystemVersion</key>
  <string>13.1</string>

  <!-- Background audio (se la radio deve continuare in background) -->
  <key>UIBackgroundModes</key>
  <array>
    <string>audio</string>
  </array>

  <!-- Categoria App Store -->
  <key>LSApplicationCategoryType</key>
  <string>public.app-category.music</string>
</dict>
</plist>

Entitlements e App Sandbox ⚡ Solo macOS – OBBLIGATORIO

🔒
App Sandbox è obbligatoria per il Mac App Store. Senza di essa Apple rifiuterà la build automaticamente. Questo è il requisito più importante che differenzia macOS da iOS.

Crea il file Platforms/MacCatalyst/Entitlements.plist con almeno questi entitlements:

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "...">
<plist version="1.0">
<dict>

  <!-- OBBLIGATORIO: App Sandbox per Mac App Store -->
  <key>com.apple.security.app-sandbox</key>
  <true/>

  <!-- Accesso a Internet (in uscita) – necessario per lo streaming radio -->
  <key>com.apple.security.network.client</key>
  <true/>

  <!-- Audio (riproduzione) – non richiede entitlement specifico oltre al sandbox -->
  <!-- Ma se usi AVAudioSession in modo avanzato, potresti aggiungere: -->
  <!-- com.apple.security.device.audio-input (solo se registri audio) -->

</dict>
</plist>

🔐 Guida agli entitlements comuni per RadioE45

EntitlementQuando aggiungerloObbligatorio
com.apple.security.app-sandbox Sempre — richiesto da Mac App Store ✅ Sì
com.apple.security.network.client L'app si connette a Internet (streaming AzuraCast) ✅ Sì per RadioE45
com.apple.security.network.server L'app accetta connessioni in entrata ❌ Non necessario
com.apple.security.device.audio-input L'app registra audio dal microfono ❌ Non necessario per radioE45
com.apple.security.files.user-selected.read-write L'app legge/scrive file scelti dall'utente ❌ Non necessario
⚠️
Ogni entitlement aggiuntivo oltre al minimo necessario richiede una giustificazione esplicita durante la revisione Apple. Aggiungi solo ciò che serve effettivamente all'app.

Build del pacchetto (.pkg) ⚡ Diverso da iOS

Su macOS l'output è un file .pkg (invece di .ipa). Hai due opzioni.

⌨️ Opzione A – Riga di comando (consigliata) Terminal

Esegui dal percorso del file .csproj:

# 1. Pubblica per Mac Catalyst in modalità Release
dotnet publish -f net9.0-maccatalyst \
               -c Release \
               -p:ArchiveOnBuild=true \
               -p:CodesignKey="Apple Distribution: TuoNome (TEAMID)" \
               -p:CodesignProvision="RadioE45 Mac AppStore" \
               -p:PackageSigningKey="3rd Party Mac Developer Installer: TuoNome (TEAMID)"

# Il file .pkg sarà in:
# bin/Release/net9.0-maccatalyst/publish/RadioE45-1.0.pkg
💡
Se il flag PackageSigningKey non produce automaticamente il .pkg, puoi usare productbuild (vedi Opzione B alternativa sotto).

🔨 Opzione B – Xcode Organizer Xcode

Apri il progetto in Xcode e seleziona come destinazione My Mac (Mac Catalyst)
Menu Product → Archive — attendi la compilazione; si aprirà l'Organizer automaticamente
Nell'Organizer seleziona l'archivio → clicca Distribute App
Scegli App Store Connect → Upload. Xcode firmerà e caricherà direttamente su App Store Connect

⚙️ Opzione C – productbuild manuale Terminal

Se hai l'app già firmata (.app) e vuoi creare il .pkg separatamente:

# Firma il bundle .app
codesign --force --options runtime \
         --sign "Apple Distribution: TuoNome (TEAMID)" \
         --entitlements Platforms/MacCatalyst/Entitlements.plist \
         "RadioE45.app"

# Crea il pacchetto .pkg firmato per Mac App Store
productbuild --component "RadioE45.app" /Applications \
             --sign "3rd Party Mac Developer Installer: TuoNome (TEAMID)" \
             "RadioE45.pkg"

Caricamento su App Store Connect ⚡ File .pkg invece di .ipa

I metodi di caricamento sono gli stessi dell'iOS, ma si usa il file .pkg e il tipo osx.

🚀 Metodo consigliato: Transporter

  1. Scarica Transporter dal Mac App Store (gratis, se non già installato)
  2. Aprilo e accedi con il tuo Apple ID sviluppatore
  3. Trascina il file RadioE45.pkg nella finestra
  4. Clicca Consegna — Transporter validerà e caricherà il pacchetto
✅
Transporter esegue anche la validazione locale prima del caricamento e ti avvisa di eventuali errori (certificati mancanti, entitlements errati, App Sandbox disabilitata) senza dover aspettare la revisione Apple.

⚙️ Metodo alternativo: xcrun altool (CI/CD) Terminal

# Carica il .pkg con API Key App Store Connect
xcrun altool --upload-app \
             --type osx \
             --file "RadioE45.pkg" \
             --apiKey  "CHIAVE_API_KEY_ID" \
             --apiIssuer "ISSUER_ID"

# In alternativa con App-Specific Password:
xcrun altool --upload-app -f RadioE45.pkg -t osx \
             -u "tua@email.com" -p "@keychain:AC_PASSWORD"
⚠️
Nota il parametro --type osx invece di --type ios. Usa sempre una App-Specific Password (generabile su appleid.apple.com), mai la password principale.

Invio alla Revisione Apple

Attendi la build su App Store Connect Dopo il caricamento, la build macOS appare sotto la sezione macOS App in App Store Connect entro 5-30 minuti. Riceverai una mail di conferma. Potrebbero volerci fino a 24h per la validazione completa.
Collega la build alla versione macOS In App Store Connect → RadioE45 → Distribuzione → macOS App → 1.0, scorri fino a "Build" e clicca + Aggiungi build. Seleziona la build macOS appena caricata.
Completa tutte le sezioni obbligatorie Controlla che non ci siano triangoli gialli ⚠️ a sinistra. Le sezioni da completare per macOS: Screenshot 1280×800, Descrizione, Note sulla versione, Informativa sulla privacy (URL), Classificazione età.
Invia per revisione Clicca "Invia per revisione" (in alto a destra nella sezione macOS). Puoi scegliere rilascio automatico dopo approvazione o manuale.
Attendi l'approvazione Apple impiega tipicamente 1-3 giorni. Per le app macOS la revisione tende ad essere leggermente più lunga rispetto a iOS per le prime submission.
🎉
Una volta approvata, RadioE45 sarà disponibile sul Mac App Store per tutti gli utenti macOS!
❌
I motivi di rifiuto più comuni per macOS: App Sandbox non abilitata, entitlements dichiarati ma non usati, screenshot nel formato sbagliato, accesso a risorse di sistema non dichiarate nel plist. Puoi rispondere nella Resolution Center di App Store Connect.

Pipeline completa macOS: dalla configurazione alla pubblicazione

📋 Checklist rapida pre-invio macOS

  • ☐ Bundle ID nel progetto corrisponde a quello nel Developer Portal (com.radio45.app)
  • ☐ ApplicationVersion incrementato rispetto all'ultimo caricamento
  • ☐ Certificato Apple Distribution presente nel Keychain
  • ☐ Certificato Mac Installer Distribution (3rd Party Mac Developer Installer) nel Keychain
  • ☐ Provisioning Profile Mac App Store Connect installato
  • ☐ Platforms/MacCatalyst/Entitlements.plist creato con App Sandbox abilitata
  • ☐ com.apple.security.network.client presente negli entitlements (per lo streaming)
  • ☐ Screenshot in formato 1280×800 px o 1440×900 px caricati
  • ☐ URL supporto e URL informativa privacy compilati
  • ☐ Classificazione età completata
  • ☐ Build .pkg caricata e collegata alla versione in App Store Connect
  • ☐ Note sulla versione compilate

🔧 Troubleshooting comune

Errore / SintomoSoluzione
"App sandbox not enabled" Aggiungi com.apple.security.app-sandbox = true in Entitlements.plist e verifica che CodesignEntitlements punti al file corretto nel .csproj
"No suitable installer signing certificate" Verifica che il certificato 3rd Party Mac Developer Installer sia nel Keychain di login. Riesegui la CSR e ricrea il certificato se necessario
Transporter: "Invalid bundle" Controlla che il Bundle ID nel .pkg corrisponda esattamente a quello registrato in App Store Connect e nel Developer Portal
"The app references non-public selectors" Problema noto con alcune versioni di .NET MAUI su Mac Catalyst. Aggiorna all'ultima versione stabile di .NET e riprova
App si chiude subito dopo l'apertura (App Sandbox) L'app tenta di accedere a risorse non dichiarate negli entitlements. Controlla i log in Console.app filtrando per il nome del processo
