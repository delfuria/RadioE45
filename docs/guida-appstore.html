Apple Developer Portal – Certificati e Provisioning

ℹ️
Vai su developer.apple.com → Certificates, Identifiers & Profiles per eseguire questi passaggi.

Flusso: App ID → Certificato → Provisioning Profile → Firma app

Registra l'App ID Vai in Identifiers → +. Scegli App IDs → App. Inserisci:
Description: RadioE45  |  Bundle ID (Explicit): com.radio45.app
Abilita le capability necessarie (es. Push Notifications, Background Audio se servono). Salva.
Genera la Certificate Signing Request (CSR) con Keychain Access Prima di creare il certificato nel portale Apple, devi generare un file CSR dal tuo Mac. Questo file contiene la tua chiave pubblica e i dati identificativi, che Apple userà per firmare il certificato.
ℹ️
Questo passaggio si esegue sul Mac prima di tornare sul portale Apple. Genera la chiave privata nel Keychain locale e produce un file .certSigningRequest da caricare.
Passaggi dettagliati in Keychain Access:
Apri Keychain Access Spotlight (⌘ Spazio) → digita Keychain Access → apri l'app.
Apri Certificate Assistant Dalla barra dei menu in alto: Keychain Access → Certificate Assistant → Request a Certificate from a Certificate Authority…
Compila il form "Certificate Information" Si apre una finestra con tre campi:
Campo Cosa inserire Esempio
User Email Address La tua email Apple ID sviluppatore delfuria@gmail.com
Common Name Il tuo nome o nome dell'organizzazione (serve solo a identificare la chiave nel Keychain) Stefano Delfuria
CA Email Address Lascia vuoto — non si invia via email ad Apple, si salva su disco — lascia vuoto —
Seleziona "Saved to disk" Nella sezione Request is, seleziona Saved to disk (non "Emailed to the CA").
Spunta anche "Let me specify key pair information" se vuoi controllare i parametri della chiave (opzionale, i default vanno bene).
Clicca "Continue" e salva il file Scegli dove salvarlo (es. Desktop). Il file si chiamerà CertificateSigningRequest.certSigningRequest.
✅
Keychain Access crea automaticamente una coppia di chiavi RSA 2048-bit nel tuo Keychain di login. La chiave privata rimane sul Mac — non va mai condivisa.
(Opzionale) Parametri chiave RSA Se hai spuntato "Let me specify key pair information", nella schermata successiva i valori corretti sono:
  • Key Size: 2048 bits
  • Algorithm: RSA
Clicca Continue.
Torna sul Developer Portal – carica la CSR:
Vai su developer.apple.com → Certificates → + e scegli Apple Distribution.
Clicca Choose File e seleziona il file .certSigningRequest salvato in precedenza. Clicca Continue.
Apple genera il certificato. Clicca Download — ottieni un file distribution.cer.
Fai doppio clic su distribution.cer — si importa automaticamente nel Keychain di login accoppiandosi alla chiave privata generata prima.
⚠️
Esegui SEMPRE la CSR sullo stesso Mac dove sviluppi. Il certificato è legato alla chiave privata in quel Keychain. Se cambi Mac, esporti il certificato con la chiave privata come file .p12 (tasto destro sul certificato in Keychain Access → Export).
Crea il Provisioning Profile App Store Vai in Profiles → +. Scegli App Store Connect.
Seleziona App ID: com.radio45.app → seleziona il certificato creato sopra → dai un nome tipo RadioE45 AppStore.
Scarica il file .mobileprovision e fai doppio clic per installarlo in Xcode.
✅
In alternativa puoi usare il Automatic Signing di Xcode: apri il progetto in Xcode, vai in Signing & Capabilities, spunta "Automatically manage signing" e accedi con il tuo Apple ID. Xcode creerà e aggiornerà certificati e profili in automatico.

Configurazione progetto .NET MAUI

Apri il file RadioE45.csproj e verifica/aggiorna le seguenti proprietà:

<!-- RadioE45.csproj -->
<PropertyGroup>
  <!-- Versione app -->
  <ApplicationVersion>1</ApplicationVersion>          <!-- Build number, incrementa ad ogni upload -->
  <ApplicationDisplayVersion>1.0.0</ApplicationDisplayVersion> <!-- Versione visibile utente -->

  <!-- Bundle ID iOS -->
  <ApplicationId>com.radio45.app</ApplicationId>

  <!-- Target framework minimo iOS -->
  <SupportedOSPlatformVersion Condition="$([MSBuild]::GetTargetPlatformIdentifier('$(TargetFramework)')) == 'ios'">15.0</SupportedOSPlatformVersion>
</PropertyGroup>

<!-- Firma per la distribuzione App Store -->
<PropertyGroup Condition="$(TargetFramework.Contains('-ios')) and '$(Configuration)' == 'Release'">
  <CodesignKey>Apple Distribution: [TuoNome] ([TEAMID])</CodesignKey>
  <CodesignProvision>RadioE45 AppStore</CodesignProvision>
  <CodesignEntitlements>Platforms/iOS/Entitlements.plist</CodesignEntitlements>
</PropertyGroup>
⚠️
Il valore di ApplicationVersion (Build Number) deve essere incrementato ad ogni caricamento su App Store Connect, anche se la versione visibile rimane la stessa. Apple rifiuta build con lo stesso numero.

📁 Verifica Info.plist (Platforms/iOS/Info.plist)

Assicurati che contenga almeno:

<key>NSMicrophoneUsageDescription</key>       <!-- Solo se usi microfono -->
<key>UIBackgroundModes</key>
<array>
  <string>audio</string>                       <!-- Riproduzione audio in background -->
</array>

Build dell'archivio (.ipa)

Hai due opzioni: da riga di comando (consigliata per automazione) o tramite Xcode.

⌨️ Opzione A – Riga di comando Terminal

Esegui dal percorso del file .csproj:

# 1. Pubblica il progetto per iOS in modalità Release
dotnet publish -f net9.0-ios \
               -c Release \
               -p:ArchiveOnBuild=true \
               -p:RuntimeIdentifier=ios-arm64 \
               -p:CodesignKey="Apple Distribution: TuoNome (TEAMID)" \
               -p:CodesignProvision="RadioE45 AppStore"

# Il file .ipa sarà in:
# bin/Release/net9.0-ios/ios-arm64/publish/RadioE45.ipa

🔨 Opzione B – Xcode Organizer Xcode

Apri il progetto in Xcode (apri il file .xcodeproj o usa Visual Studio for Mac)
Seleziona il target come Any iOS Device (arm64) nella barra in alto
Menu Product → Archive — Xcode compilerà e aprirà l'Organizer
Nell'Organizer, seleziona l'archivio e clicca Distribute App → App Store Connect
💡
Per .NET 8 sostituisci net9.0-ios con net8.0-ios nel comando dotnet publish.

Caricamento su App Store Connect

Hai tre modi per caricare il file .ipa:

Tre metodi per caricare l'IPA su App Store Connect

🚀 Metodo consigliato: Transporter

  1. Scarica Transporter dal Mac App Store (gratis)
  2. Aprilo e accedi con il tuo Apple ID sviluppatore
  3. Trascina il file RadioE45.ipa nella finestra
  4. Clicca Consegna — Transporter caricherà e validerà il binario

⚙️ Metodo alternativo: xcrun altool (CI/CD) Terminal

# Carica con App-Specific Password
xcrun altool --upload-app \
             --type ios \
             --file "RadioE45.ipa" \
             --apiKey  "CHIAVE_API_KEY_ID" \
             --apiIssuer "ISSUER_ID"

# In alternativa con username/password App-Specific:
xcrun altool --upload-app -f RadioE45.ipa -t ios \
             -u "tua@email.com" -p "@keychain:AC_PASSWORD"
⚠️
Usa sempre una App-Specific Password (generabile su appleid.apple.com), mai la password del tuo Apple ID principale.

Invio alla Revisione Apple

Attendi la build su App Store Connect Dopo il caricamento, la build appare in TestFlight entro 5-30 minuti (a volte più). Riceverai una mail di conferma. Potrebbero volerci fino a 24h per il completamento della validazione.
Collega la build alla versione In App Store Connect → La mia app → RadioE45 → Distribuzione → 1.0, scorri fino a "Build" e clicca + Aggiungi build. Seleziona la build appena caricata.
Completa tutte le sezioni obbligatorie Controlla che non ci siano triangoli gialli ⚠️ a sinistra. Le sezioni più comuni da completare: Screenshot, Descrizione, Note sulla versione, Informativa sulla privacy (URL), Classificazione età.
Invia per revisione Clicca "Invia per revisione" (in alto a destra). Se vuoi, abilita il rilascio automatico dopo l'approvazione, oppure imposta il rilascio manuale.
Attendi l'approvazione Apple impiega tipicamente 1-3 giorni per le prime submission, meno per gli aggiornamenti. Ricevi notifiche via e-mail sullo stato.
🎉
Una volta approvata, l'app sarà disponibile sull'App Store per tutti gli utenti!
❌
Se Apple rigetta l'app, riceverai un'email con i motivi. Puoi rispondere direttamente nella Resolution Center di App Store Connect, correggere e rinviare.
